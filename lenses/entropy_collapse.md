## Entropy Collapse

Security mechanisms often rely on values assumed to be unpredictable.

When implementation details constrain those values — even unintentionally —
the system may continue to operate as if randomness exists, while in reality the space has been drastically reduced.

---

### Observation

Small or predictable domains do not need to be broken —
they can be exhausted.

When unpredictability is assumed but not enforced,
security degrades silently.

---

### Takeaway

Randomness is not a property of intention.
It is a property of implementation.
